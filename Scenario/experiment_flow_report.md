# 📊 로컬 AI 기반 시나리오 어시스턴트: 데이터 추출 고도화 및 파인튜닝 연구 보고서

## 1. 연구 배경 및 문제 제기 (Why Local AI?)
시나리오 작성 보조(세계관 관리) 프로그램 개발 과정에서 초기에는 상용 대형 언어 모델(Claude 3.5 Haiku 등) API를 활용하여 세계관 데이터(인물, 장소, 플롯)를 추출하였습니다. 그러나 상업적 서비스 관점에서 다음과 같은 치명적인 한계점이 발견되었습니다.

* **보안 문제 (Security):** 미출간된 작가의 독창적인 아이디어와 세계관 원문이 외부 클라우드 서버로 전송되어 유출될 위험성.
* **토큰 비용 (Cost):** 장편 소설 연재의 특성상 텍스트 입력량이 매우 방대하며, 지속적인 API 호출은 막대한 서버 유지 비용을 발생시킴.

따라서 100% 오프라인 환경에서 동작하며 보안을 보장하는 **로컬 소형 모델(sLLM)**의 도입이 필수적임을 확인하였습니다.

## 2. 초기 비교 실험 (Baseline Experiment)
로컬 모델이 상용 모델을 대체할 수 있는지 확인하기 위해, 1930년대 한국 단편 소설 데이터셋(200 샘플)을 활용하여 비교 실험을 진행했습니다.
두 모델에게 **동일한 원문**과 **동일한 프롬프트**를 부여하고, 지정된 복잡한 JSON 스키마(인물, 사건, 장소 및 동적 세부 속성)로 엔티티를 추출하도록 지시했습니다. 
(※ Claude 3.5 Haiku가 완벽하게 추출한 정답 데이터를 **Pseudo-Gold Standard**로 간주하여 로컬 모델의 성능을 측정함)

## 3. 제로샷(Zero-shot) 로컬 모델의 한계 확인
실험 결과, 파인튜닝 되지 않은 로컬 베이스 모델(Gemma 26B)은 다음과 같은 심각한 성능적 한계를 노출했습니다.

* **형식 파괴:** 복잡한 계층의 JSON 스키마를 100% 준수하지 못하고 구조가 깨지는 현상 발생 (프론트엔드 DB 연동 불가).
* **정보 누락 및 환각:** 원문에 존재하는 주요 개체를 추출하지 못하거나(Recall 저하), 세부 묘사(Details)의 깊이가 현저히 얕음. 

## 4. 로컬 모델 파인튜닝 (SFT via QLoRA)
로컬 모델의 한계를 극복하기 위해, 상용 모델(Claude)이 추출했던 양질의 정답 데이터를 학습 데이터셋으로 가공하여 로컬 모델에 직접 학습(Supervised Fine-Tuning)시키는 전략을 채택했습니다.

* **학습 기법:** QLoRA (단순 형태 암기를 넘어 논리적 인과관계를 학습하도록 `r=32`의 높은 Rank 부여)
* **Target Modules:** All Linear (`q, k, v, o, gate, up, down_proj` 등 MLP 레이어 전반까지 깊게 튜닝하여 JSON 구조 및 도메인 지식 강제)
* **Epochs:** 4 (과적합 방지 및 포맷 완벽 숙지를 위한 최적화)

## 5. 파인튜닝 후 성능 개선 지표 (Evaluation Metrics)
파인튜닝된 소형 로컬 모델(Gemma 4B)과 기존 베이스 모델(Gemma 26B)의 성능 지표를 동일한 200개의 평가 데이터로 비교한 결과, 파라미터 수가 훨씬 적음에도 불구하고 **파인튜닝된 소형 모델이 압도적으로 향상된 성능**을 보여주었습니다.

| 평가 지표 (Metric) | 베이스 모델 (Gemma 26B Base) | 파인튜닝 모델 (Gemma 4B SFT) | 성능 변화 |
| :--- | :---: | :---: | :--- |
| **스키마 준수율 (Schema Consistency)** | 53.00% | **100.00%** | 에러율 0% 도달, 완벽한 호환성 확보 |
| **총 추출 개체 수 (Total Entities)** | 665 | **1,342** | 누락되었던 2배 이상의 개체 식별 |
| **개체당 평균 세부 묘사 (Avg Details)** | 2.04 | **3.87** | 인물/배경 묘사의 뎁스(Depth) 189% 증가 |
| **F1-Score (Claude 대비 정확도)** | 0.1739 | **0.5124** | Claude 모델 수준에 3배 더 근접하게 추적 성공 |

### 💡 결론 요약
초기 로컬 모델은 사용이 불가능한 수준의 에러율(47%)과 극히 낮은 정확도를 보였으나, 정교하게 설계된 파이프라인으로 **파인튜닝(SFT)을 진행한 결과 모델의 체급(26B -> 4B)을 줄였음에도 불구하고 스키마 준수율 100%를 달성하고 모든 정보 추출 지표(F1-Score, 추출량 등)가 비약적으로 상승**하였습니다. 이는 상용 API 의존성을 끊고 자체 로컬 AI 서버 구축이 가능함을 입증한 결과입니다.
