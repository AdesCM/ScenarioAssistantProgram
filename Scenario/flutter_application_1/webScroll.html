초기 코드의 한계점

구현 3: image_picker 활용

핵심 기술: image_picker 패키지

구현: 의존성 추가, 모델 imageUrl 필드, 2단 레이아웃, _pickImage, _buildImageWidget

코드 예시:

// models/character.dart - 선택적 imageUrl
Character({ /* ..., */ String? imageUrl })
  : imageUrl = imageUrl ?? '기본_이미지_URL';

// screens/character_detail_screen.dart - 이미지 선택 함수
Future<void> _pickImage() async {
  final picker = ImagePicker();
  final pickedFile = await picker.pickImage(source: ImageSource.gallery);
  if (pickedFile != null) {
    setState(() { _pickedImage = File(pickedFile.path); });
  }
}

주요 발생 오류 및 해결 (1/2)

  • 의존성 오류: pubspec.yaml 패키지 누락 → flutter pub get
  • YAML 들여쓰기 오류: dependencies: 위치 → 올바른 레벨로 수정
  • Import 오류: package.flutter → package:flutter (콜론 사용)
  • VS Code 작업 공간 오류: 하위 폴더 열기 → 프로젝트 루트 폴더 열기
14 / 15

주요 발생 오류 및 해결 (2/2)

  • 상태 관리/빌드 오류:
    • LateInitializationError (_tags) → 즉시 초기화 (= [])
    • ChangeNotifier 오류 → with ChangeNotifier, 분석 서버 재시작
    • 다른 드라이브 빌드 오류 → 프로젝트 C: 이동, flutter clean
  • Flutter 규칙/Lint 오류: const, key 누락 → 자동 수정(Quick Fix) 활용
  • 위젯 매개변수 오류: Chip의 onPressed 부재 → InputChip 사용
  • UI 멈춤: build 내 필터링 → 로직 분리(_filterElements), 상태 변수 사용
