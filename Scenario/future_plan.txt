# 차후 계획 및 시스템 아키텍처 (Future Plan & Architecture)

본 문서는 Scenario Assistant 시스템의 중간 단계 파이프라인 정리 및 향후 개발 플랜을 문서화한 것입니다. 프로젝트 진행 시 참고 자료로 활용됩니다.

## 1. 현재 데이터 플로우 및 진행 상황 검토
**[처리 파이프라인]**
유저 입력 텍스트 -> 로컬 모델 파싱(JSON 추출) -> Microsoft GraphRAG 연동을 통한 무결성 검사 -> 시스템 데이터 저장 (메타데이터 반영)

- **현황**: 현재 로컬 모델을 통해 텍스트 내에서 인물, 장소, 사건 등의 객체를 추출하고 JSON 및 그 외의 포맷으로 정형화하는 작업이 구현 중임.
- **병목 이슈**: 프롬프트 엔지니어링에만 의존하는 방식으로는 복잡한 맥락에서의 추출 누락이나 JSON 양식 파괴(Halucination)와 같은 안정성 문제가 일부 발생함. 따라서 AI 모델의 직접적인 성능 향상이 전체 정확도에 지대한 영향을 미침.

## 2. 최종 구현 목표: 지식 그래프 시각화 (Visualization UI)
- 단순히 객체를 요약해서 나열하는 기존 방식을 넘어서, 사용자에게 직관적인 연결 구조를 시각화하는 것을 목적으로 함.
- **UI 구현계획**: 
  1. 개체 데이터 윈도우 (예: '홍길동' 데이터 창)에서 기존 [요약] 버튼 옆에 **[시각화] 버튼**을 추가로 배치.
  2. 버튼 클릭 시 해당 엔티티를 중심 노드로 하는 시각적인 마인드맵(지식 그래프)이 랜더링 되어 화면에 표출됨 (RAG 연계망 시각화).
  3. 우측에 별도의 **필터 팝업 컨트롤러**를 띄워, 유저가 원하는 정보만 선택해서 볼 수 있게 함 (ex: 인물 필터 / 플롯 필터 / 장소 필터).
  4. 복잡한 관계망 표현: 장소가 플롯에 종속되거나 관여될 경우, 일렬로 무작위 나열되는 것이 아닌, **플롯 노드를 거쳐 파생(브랜치)되는 형태**로 직관적 계층 관계를 구성.

## 3. 차후 실험 및 모델 고도화 계획
향후에는 추출 정확도를 높이기 위한 실험적 접근 방식과 자체 SFT(지도 학습) 모델 파인튜닝을 적용할 계획.

- **학습 데이터 (Dataset) 구성**:
  * 모델이 세계관 정보를 원활히 인식하도록 Kaggle의 Fantasy Worldbuilding Dataset 등의 데이터를 확보.
  * 퍼블릭 소설 데이터 중 약 15,000 ~ 20,000건의 '원본 텍스트 - 구조화된 JSON 매핑 데이터'를 구성.
- **파인튜닝 (Fine-Tuning) 전략**:
  * Llama-3 시리즈, 혹은 Qwen 구조의 로컬 모델을 Base 모델로 지정.
  * 정보추출에 맞게 구성된 데이터셋을 통해 LoRA 등의 파라미터 효율적 학습 기법으로 구조적 추론 능력을 튜닝.
- **타겟 지표 개선도 (Target Metrics)**:
  * 특정 포맷(JSON) 미준수율: 15% 이상에서 **1% 미만**으로 하락시켜 시스템 런타임 오류 방지.
  * 개체 관계 추출(Entity & Relation Extraction) F1-Score: Zero-shot 환경 기준 추정치 **0.65 -> 0.92 이상**으로 대폭 상향.
- **비즈니스 임팩트 (Impact)**:
  * 첫 단계(로컬 추출)의 정확도가 올라가면, 후단에서 이루어지는 MS GraphRAG의 환각 검증/무결성 평가 단계에서의 에러 검출 비율이 비례하여 낮아짐. 
  * 결과적으로 유저가 텍스트를 입력하고 데이터베이스에 결점 없이 반영되는 성공률을 최적화할 수 있음.
