1안. 시나리오 라이팅 서포트 프로그램

01. 선정 이유

  • 창작의 휴먼 에러 최소화
    • 방대한 세계관 설정과 복잡한 인물 관계 속에서 필연적으로 발생하는 '설정 오류'를 사전에 방지
  • 관계의 시각화 필요성
    • 수백 페이지의 텍스트만으로는 파악하기 힘든 세부 설정을 마인드맵/지식 그래프로 한눈에 파악
  • 시장성 고려
    • 웹소설, 서브컬처 게임 등 스토리텔링 콘텐츠 시장 확대로 인한 강력한 지능형 에디터 보조 툴 수요 존재
  • 레퍼런스 및 기획 배경
    • 특정 상용 레퍼런스를 벤치마킹하기보다는, 지난 학기 진행했던 프로젝트의 연속선상에서 완성도를 고도화하며 실질적으로 AI와 연동시키는 것이 목적

02-1. 구현 검증 (AI 파이프라인 및 데이터 추출)

  • 핵심 기술: 개체명 인식(NER) 및 관계 추출(RE)
    • Transformer 기반의 언어 모델을 활용해 텍스트 내 엔티티(인물, 장소, 소속)를 분리
  • 특화 LoRA 파인튜닝
    • 단순 언어모델은 복잡한 판타지/무협 용어를 오용할 수 있음.
    • 오픈소스 LLM (Llama-3, Qwen 등)에 특정 장르 데이터를 LoRA로 어댑터 튜닝하여, "A가 B를 배신했다"는 문맥을 정확히 파악하도록 학습
    • 최종적으로 추출된 관계 데이터를 정형화된 JSON 포맷으로 강제 출력 하도록 프롬프트 엔지니어링 수행

    • 단순 리스트 나열이 아닌, 사용자가 노드를 드래그 앤 드롭으로 재배치하고 연결선을 직관적으로 확인할 수 있는 인터랙티브 UI 제공

02-2. 구현 검증 (상용 LLM API 활용 대체 방안)

  • 직접 모델 학습의 대안

    • 하지만 '프로그램 기능의 완성' 자체에 초점을 둔다면, 훨씬 효율적인 접근법을 선택할 수 있습니다.

    • 최근의 강력한 상용 LLM이나 Claude 3.5 모델 등은 이미 방대한 문맥 파악 및 개체 간의 관계 추론 능력을 충분히 학습한 상태입니다.
    • 따라서 별도의 자체 모델 파인튜닝이나 데이터셋 구축 과정 없이, 정교한 프롬프트 엔지니어링과 API 호출만으로도 요구하는 형태(JSON 관계도)를 충분히 추출하고 웹 프론트엔드와 연동할 수 있습니다.(2025-2 실험 수행)

04. 최종 목표 (UI/UX 시나리오)

  • 입력 단계: 웹 기반의 에디터 화면 좌측에 사용자가 대본 혹은 소설 텍스트 뭉치(수천 자 분량)를 복사-붙여넣기 후 '분석' 실행.
  • 시각화 대시보드: AI 파서를 거친 데이터가 '세계관'별로 정리되어 해당 세계관 데이터에서 '마인드맵 트리'UI를 선택 시 등장인물 간의 관계선(적대, 우호, 가족 등) 표현.

2안. 판타지 경주 시뮬레이터

01. 선정 이유

  • 데이터 기반의 설계로 시뮬레이션 장르의 깊이 확보
    • 단순한 난수 발생기(RNG) 의존형 게임을 지양하고, "현존하는 실제 데이터"를 기반으로 엔진에 접목하려는 기획적 시도.
  • 플레이어 몰입감 및 인과성 부여
    • 말을 '그리폰' 같은 판타지 생물로 치환하면서, 생물학적 특징(예: 깃털 방수력, 날개 면적)이 실제 날씨 데이터와 어떻게 상호작용하는지 논리적 인과성을 제공해 유저가 납득할 수 있는 게임 규칙 형성.
  • 개인적인 목표
    • 플레이어의 선택이 준비 -> 실시간의 연속된 선택 -> 결과가 아닌, 준비 -> 결과로 이어지는 시뮬레이션 장르의 게임을 경험해보고 싶은 목적

2안. 판타지 경주 시뮬레이터

02-1. 구현 검증 (학술적 생물학 이론의 변수화)

  • 현실 생물학 논문의 게임 규칙화
    • 예측을 정교하게 만들기 위해 단순 경마 스탯(근력, 속도, 지구력, 지능) 외에 추가 변수 설계가 필요.
  • 조류 꼬리샘 연구 활용
    • "조류의 미지선에서 분비되는 기름기 성분이 깃털의 방수 효과와 비행 에어로다이나믹스(공기 저항 감소)에 영향을 미친다"는 실제 연구 논문을 게임 시스템에 접목.
    • 이를 통해 '미지선 발달 수치(유분기 정도)'가 높은 교배종은 비 내리는 트랙 환경에서 스피드 디버프를 무효화하는 형태의 정교한 수학적 보정 계수로 알고리즘에 반영.

2안. 판타지 경주 시뮬레이터

02-2. 구현 검증 (AI 예측 모델 및 아키텍처)

  • 회귀 추론(Regression) 파이프라인
    • 수집된 다차원 데이터(기수 스탯, 환경 변수 등) 기반으로 XGBoost 또는 딥러닝(MLP) 회귀 모델 학습 (Scikit-learn, PyTorch 활용)
    • 경기 결과 순위 분류(Classification)뿐만 아니라, 예상 도달 시간(Lab time)의 연속적 수치를 예측.
  • 시각화 및 엔진 연동 (게임화)
    • 파이썬 백엔드(FastAPI)에서 해당 회차의 판정 데이터(각 생물의 체크포인트별 속도 곡선)를 JSON Array로 클라이언트에 전송.
    • 동적 시뮬레이터(Unity 3D / WebGL)는 결과값을 받아 트랙상의 생물 애니메이션에 선형 보간(Interpolation)을 줘서 레이싱 연출을 실행.

2안. 판타지 경주 시뮬레이터

03. 데이터셋 확보 방안 (난이도: 중상)

  • 기반이 되는 경마 대규모 데이터 확보
    • Kaggle JRA (Japan Racing Association) Dataset: 1986년부터 쌓인 방대한 과거 트랙 데이터, 순위, 마체중 변수를 머신러닝 기초 훈련 셋으로 사용
    • Korea Horse Racing (한국마사회 KRA) 등 지역별 데이터를 보충재로 활용.
  • 데이터 튜닝(Feature Engineering)
    • 위의 현실 경마 데이터 속 '말(Horse)'을 '비행 조수(Bird/Griffon)'로 치환하기 위한 추가 칼럼(날개 면적, 꼬리샘 활성도) 파생 변수 생성이 이 프로젝트 구현의 최대 관건.

2안. 판타지 경주 시뮬레이터

04. 최종 목표 (게임 데모)

  • 유저가 4~8마리의 육성된 판타지 생물(스탯과 외형이 다름)의 출전 명단을 구성하고, 날씨/풍향/습도를 임의로 조작.
  • AI의 역할은 실시간 레벨스케일링을 자동화 하는 것. 현재 캐릭터의 능력치와 환경 요소를 기반으로 레이스에 참여할 다른 주자들의 스탯을 생성하는 것.
  • "Start Game" 버튼을 누르면 정해진 수치와 수식에 따른 결과를 순식간에 계산하고, 브라우저 화면에서 결과에 맞춰 달리는 생물들을 중계하듯 보여주며 레이스를 마친 후 예측 근거(승리 요인)를 함께 출력함.

3안. 레인보우 식스 시즈: 맵 자동 생성

01. 선정 이유

  • e스포츠 밸런스 해결책 도출
    • R6 시즈와 같은 전술 FPS에서 벽/바닥의 파괴 및 강화 구조는 프로 리그 전략의 핵심. 공격과 수비의 밸런스가 한쪽으로 쏠리면 게임의 핵심 재미가 사라짐.
  • AI 레벨 디자인의 공학적 접근법 시도
    • 숙련된 레벨 디자이너의 경험과 직관에 100% 의존하던 맵 기획을, '맵의 위치의 킬/데스 로그 및 승률 메타데이터', '데이터를 통한 밸런스 지점 생성' 등의 데이터 기반으로 유의미하게 보조 혹은 생성해낼 수 있는지 컴퓨터공학적으로 증명.

3안. 레인보우 식스 시즈: 맵 자동 생성

02-1. 구현 검증 (PCG-ML 기반 생성 아키텍처)

  • 절차적 머신러닝 아키텍처 (PCG-ML)
    • 맵 데이터를 단순 벡터나 3D 오브젝트가 아닌, 평면 탑다운 형태의 '그리드 이미지(Grid Image)' 기반으로 전환하여 다룸.
  • 조건부 이미지 생성망 도입
    • 사용자가 입력한 목표 승률 목표값과 색상 기반 라벨링 이미지를 훈련 데이터로 활용.
    • Pix2Pix (Image-to-Image Translation) 혹은 컨트롤넷을 응용하여, 주어진 조건에 맞춰 방어 구역의 철골벽(빨강), 파괴가능벽(파랑), 바닥재(노랑) 픽셀의 최적 배치 이미지를 합성해냄.
    • 게임 구조상 필수적인 외벽 진입로나 계단 위치 등은 삼각형의 꼭짓점 형태의 핫스팟 위주로 생성.

3안. 레인보우 식스 시즈: 맵 자동 생성

02-2. 구현 검증 (논리적 무결성 - Playability)

  • "그럴싸한 쓰레기" 방지 검증 로직
    • 이미지 생성 AI는 픽셀 텍스처를 따라 그릴 뿐, 그곳이 통과 가능한 '문'인지 막힌 '벽'인지 인지하지 못합니다. (환각 현상)
  • A* (A-Star) 길찾기 알고리즘 교차 검증
    • AI가 뱉어낸 맵 평면도 이미지를 다시 행렬 구조로 파싱 한 뒤, 스폰 포인트에서 목표 지점까지 길찾기 알고리즘을 돌려봅니다.
    • 만약 완전히 막혀있어 도달 불가능하다면, 해당 결과물을 폐기하고 가중치를 미세 조정하여 다시 생성 하는 안전 장치 파이프라인 필수. 특히 방과 방을 통해 모든 공간을 이동할 수 있는지를 확인해야함.

3안. 레인보우 식스 시즈: 맵 자동 생성

03. 데이터셋 확보 방안

  • 보유 데이터의 '시맨틱 세그멘테이션' 맵핑
    • 보유 중인 모든 공식 탑뷰 도면 이미지에 **색상 코딩(Color-coding)**을 적용하여 AI가 직관적으로 학습할 수 있도록 전처리
    • 예: ■ 철골벽, ■ 소프트월, ■ 바리케이드/문, ■ 파괴형 바닥재
  • 학습 난이도의 획기적 하락
    • 일반적인 텍스트 라벨링이 아닌 "색상 블록 이미지 자체"를 정답지로 제공하게 되어, 이미지 생성 AI라서 기하학적 구조 특징을 훨씬 정확하고 빠르게 학습할 수 있습니다. 가장 큰 데이터 입력 병목 지점이 완벽히 해소됩니다.
  • 플레이 메타데이터 보조 결합
    • Kaggle R6 Siege Ranked Dataset 의 승률/픽률 수치를 각 도면과 짝지어 조건 텍스트로 보조 입력.

3안. 레인보우 식스 시즈: 맵 자동 생성

04. 최종 목표 (도면 제너레이터)

  • 조건 입력: 목표로 하는 방어측 승률 49.8%, 소프트월(파괴 가능한 벽) 비율 30% 증가 등의 환경 지표를 UI를 통해 콘솔에 입력.
  • 파이프라인 가동: AI 생성 도구가 이미지 생성 → 검증 도구가 플레이 가능 여부 승인.
  • 최종 산출: 모든 검증을 마친 새로운 전술 맵 평면도 이미지가 출력되며, 핫스팟(교전 예상 구역)이 히트맵 형태로 도면 위에 투영되어 시각적으로 나타남.

4안. 게임 유저 평가 요약 및 분석기 (RAG)

01. 선정 이유

  • 라이브 서비스 게임의 생명줄, 초기 대응 속도
    • 각종 커뮤니티(아카라이브, 디시인사이드, 인벤, 레딧)에서 쏟아지는 수만 건의 피드백을 인간 운영진이 실시간으로 개별 분석하는 것은 불가능합니다.
  • 단순 감성 분석을 넘은 '인과 분석'
    • 기존의 긍정/부정(Positive/Negative) 그래프만 띄워주는 감성 분석 툴은 실무적 가치가 낮습니다.
    • 왜 부정적인가? "이번에 신규 도입된 B 강화 시스템의 재화 소요량이 기존 무과금 기준선(천장)을 침해했기 때문"이라는, 게임 고유의 지식 기반 분석이 운영진에게 필요합니다.
  • 숨겨진 감정 찾기
    • 유저들은 부정적, 긍정적 요소를 직접적으로 표현하기도 하나, 그 외에도 다양한 요소가 유저에 표현에서 확인됩니다. "수익이 줄었다", "이제는 꿀이 아니다" 같은 비수사적 표현으로도 유저들의 감정을 확인할 수 있어야합니다.

4안. 게임 유저 평가 요약 및 분석기 (RAG)

02-1. 구현 검증 (RAG 파이프라인 아키텍처)

  • RAG (Retrieval-Augmented Generation, 검색 증강 생성) 핵심
    • LLM(대형 언어 모델)의 고질적 환각(Hallucination)을 막기 위해, LLM에 질문을 던지기 전 '정답 참고서'를 쥐어주는 기술.
  • Vector Database 구축 (지식의 수치화)
    • 대상 게임의 과금 모델(BM) 기획서, 캐릭터 상성표, 패치노트 전문을 임베딩(Embedding) 모델 처리를 거쳐 벡터 형태로 ChromaDB나 Pinecone 같은 벡터 데이터베이스에 저장합니다.
    • 크롤링된 유저의 불만 글의 문맥 유사도를 비교해 해당 DB에서 가장 관련 깊은 게임 시스템 정보를 먼저 검색(Retrieval)해 엮습니다.

4안. 게임 유저 평가 요약 및 분석기 (RAG)

02-2. 구현 검증 (커뮤니티 은어 이해 방안: RAG vs 파인튜닝)

  • 전문 용어 및 밈(Meme) 언어 장벽 해소
    • "혜자다", "천장 쳤다", "사료 좀 뿌려라" 같은 국내 게임 커뮤니티 특유의 은어 속뜻을 정확하게 파악해야 완벽한 인과 분석이 가능합니다.
  • 접근법 1: 도메인 어댑테이션 (LoRA 파인튜닝)
    • 은어 말뭉치를 구축해 LLM 자체에 학습시키는 방식으로, 모델이 게임 특유의 화법과 뉘앙스를 스스로 체화하도록 만듭니다.
  • 접근법 2: 은어 사전 기반 RAG 활용 (유연한 대체 방안)
    • 신규 밈이 실시간으로 쏟아지는 커뮤니티 특성상 매번 모델을 재학습하는 것은 비효율적일 수 있습니다.
    • 이를 극복하기 위해 '실시간 커뮤니티 은어 사전'을 DB에 저장하고, 유저 텍스트 내 특정 단어 발견 시 RAG로 그 뜻을 검색하여 LLM에 프롬프트로 쥐어주는 방식(지식 주입)이 훨씬 유지보수에 유리하고 직관적입니다.
